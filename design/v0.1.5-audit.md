# v0.1.5 audit record — "the promise becomes checkable"

Distilled record of the multi-agent audit that scoped the v0.1.5 release.
Method: 8 read-only lens assessors (research-grade, ui-ux, maintainability,
performance, installer-cli, gate-machinery, ci-supply-chain, adversarial
determinism) → 71 raw gaps deduped → top 30 adversarially verified by
independent skeptic agents instructed to refute (each read the cited files) →
completeness critic → 3 release designers (enforcement-first, exemplar-first,
release-engineering). 42 agents total. 27 gaps survived verification; 9 were
refuted on inspection. The release plan took the three designs' consensus
core plus the exemplar raises for the two weakest-scoring benchmarks.

The benchmark: "an agent operating within this harness will deterministically
produce a codebase that is (1) a research-grade artifact, (2) benchmark-
setting UI/UX, (3) world-class maintainable, (4) world-class performant."
*Deterministically* means gate/hook/CI-enforced, not exemplified or prose.

## Scores at v0.1.4 (0–10 vs world-class)

| Lens | Score | Headline finding |
|---|---|---|
| performance | 4.5 | one hard-coded SSR proxy subject, ~12× slack, no size/latency baselines |
| ui-ux | 5.0 | no write UX at all; naive palette; primitives rule prose-only; zero forced-colors |
| research-grade | 5.5 | citations only had to *resolve*; any `https://` grounded; dual license unverifiable |
| maintainability | 5.5 | machinery exempt from its own bar; aggregate coverage lets 0% files ride |
| installer-cli | 6.5 | seedOnInitOnly hand-maintained, no completeness gate |
| gate-machinery | 6.5 | strong fail-closed discipline; the above are the residual green-but-bad paths |

## Confirmed gaps → disposition

Shipped in v0.1.5 (the item that closed it):

- Semantic citation mismatch uncaught → provenance group-match + overrides (A2)
- Any `https://` grounds a citation → citation-domains host allowlist (A2)
- Not REUSE/SPDX compliant → REUSE.toml + LICENSES + blocking `reuse lint` (B1)
- Machinery exempt from its own bar / installer unlinted → root eslint +
  checkJs + knip, blocking, 11-site honest ratchet (B2)
- Aggregate-coverage test-washing → per-file floors + `diff-coverage` Stop step (A3)
- perf-budget single-subject / agent screens unmeasured (two findings merged)
  → `subjects[]` + dense-feature-must-be-measured closure (A4)
- ~10× perf slack, no committed baseline → gzip ratchet vs committed baseline (C4)
- No binary/installer size budget → nightly NSIS size assert (C4)
- No user-perceived latency enforcement → CI-only interaction lane:
  TTI + rAF-bracketed arrow latency + longtask ceiling, busy-loop canary (C5)
- No write/mutation UX exemplar → optimistic create-note slice + held-POST e2e (C1)
- Palette far below tier → fuzzy ranking + groups + keys + recents + contextual (C2)
- Primitives rule advisory → `controlPrimitives` styleguide scan, gate-red (A6)
- No forced-colors / AAA coverage → forced-colors layer + lane; 7:1 computed
  tier on primary reading pairs, both themes (C3)
- Reviewer read-only is prose → docs-sync roster sub-check, fail-closed parse (A5)
- seedOnInitOnly completeness ungated → check-seeded-migrations selftest gate (A1)
- No adoption ramp (critic: the release itself would red 0.1.4 consumers)
  → `baseVersion` + `rampNote`, shape-conditional activation, v0.1.4
  full-validate update-skew leg as a per-PR invariant (A1)
- Wall-clock in the Stop chain breaks determinism (critic, on the audit's own
  first draft) → the interaction lane is CI-only by construction (C5)
- gates-catalog drift ungated → docs-sync catalog lockstep (A1)

Deferred (recorded in the plan, revisit for v0.1.6):

- Default-on mutation testing (incremental-runtime story needed)
- General DAL query-plan coverage gate (beyond the RLS list pattern)
- Duplication detection + max-lines in the CONSUMER config (needs its own ramp)
- Installer atomicity/rollback sentinel
- SLSA verification on the advertised `npx github:` path (npm publish decision)
- README quantitative-claims recomputation gate
- Interpreter-write (`node -e`) bash-guard hardening (tamper evidence is
  already honestly scoped as evident-not-proof)

## Refuted claims (kept so they are not re-litigated)

Nine findings did not survive skeptical verification, chiefly: the npm graph
IS pinned where it matters (catalog + lockfiles in scaffolds), error surfaces
already lead with human-authored copy, the perf subject's full-grid render is
a deliberate relative canary (its header says so), hit targets meet the AA
floor by design, and doctor's exit codes already serve CI. Full texts live in
the session transcript; titles: lockfile pinning, eval statistical rigor,
motion-token budget, iconography system, raw error strings, hit-target size,
API-doc coverage gate, full-grid perf subject, doctor --json.
