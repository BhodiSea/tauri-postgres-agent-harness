# v0.1.6 audit record — "the four pillars become floors"

Distilled record of the multi-agent audit that scoped the v0.1.6 release, and of
the adversarial reviews run *during* the build.

Method (scoping audit): 8 read-only lens assessors (research-grade, ui-ux,
performance, maintainability, trust-root, installer-cli, gate-machinery,
adversarial-determinism) → raw gaps deduped to 30 → one independent skeptic per
finding instructed to *refute* (each read the cited files) → completeness critic
→ grade synthesis. ~50 agents. 31 findings survived verification; 2 were refuted
outright and ~9 were materially narrowed — the narrowed scopes were respected in
the plan, not the original overstatements.

The benchmark is unchanged from v0.1.5: "an agent operating within this harness
will deterministically produce a codebase that is (1) a research-grade artifact,
(2) benchmark-setting UI/UX, (3) world-class performant, (4) world-class
maintainable." *Deterministically* means gate/hook/**or blocking-CI**-enforced —
prose, prompts and exemplars are advisory and do not count.

## Scores at v0.1.5 (0–10 vs world-class, adversarially verified)

| Pillar | Grade | Headline finding |
|---|---|---|
| research-grade | C (5.0) | "citation justifies" held only for `[corpus:]` cites, only on fresh installs, only for 6 hard-wired classes; artifact had no LICENSE/CITATION.cff |
| ui-ux | B− (6.0) | flagship write-UX exemplar 401'd against its own server; route-state/focus/reflow existence-only or home-route-only; no status color; no i18n |
| performance | C+ (5.5) | only agent-time signal was an effect-free SSR canary; real Tauri/IPC/Rust cost, memory, added dense screens, general query plans all unmeasured |
| maintainability | B (6.5) | per-file coverage floors real, but coverage measures execution not assertion; zero duplication detection; unbounded DTO strings; machinery red under its own bar |

Two structural leaks recurred across every pillar: (1) gates could be disarmed
while the whole board stayed green (a symlink walked through the write-guard; a
committed `baseVersion` downgrade turned live checks into advisory NOTEs in CI);
(2) enforcement closed over the two shipped exemplars, not the agent's own added
screens, queries, IPC handlers and research constants.

## Confirmed gaps → disposition

Shipped in v0.1.6 (workstream that closed it):

- Symlink shadows the write-guard → `realpath`-resolve before matching (W1/G01)
- Manifest has no root of trust; `baseVersion` downgrade disarms CI → git-monotonic
  baseVersion + consumer-CI regen from the SHA-pinned tag + metadata in
  gate-integrity (W1/G03)
- `tsconfig(.base).json` weakenable by nothing → added to protected files (W2/G07)
- Interpreter/patch writes bypass the guard; workflows/escape-lists untamper-evident
  → bash-guard classes + gate-integrity surface + escape-list hashing (W2/G02,G08)
- Flagship desktop slice sends no `Authorization` → one authenticated api-client +
  a blocking real-server integration lane (also fixed: server had **no CORS**) (W3/C01)
- No danger/success token; error toast identical to confirmation → computed
  status-color tokens + status-channel boundary scan (W9/G19)
- Route state/focus/reflow exemplar-only → EmptyState/Skeleton render check,
  per-route focus walk, 640×480 reflow sweep over ROUTES (W10/G20,G21,G23)
- Single-locale English → turn-fatal i18n seam + gate + pseudo-locale/RTL lane (W11/G22)
- Citation duty only 6 hard-wired classes; ~25 groups-less universal justifiers;
  semantic checks NOTE forever → consumer decision-groups, groups-required hard
  error, auto-graduate (W7/G27,C02,G26)
- Artifact not citable; README/CHANGELOG figures hand-authored and contradictory;
  corpus self-certifying → LICENSE/CITATION.cff + claims gate + network fidelity
  job (both the claims gate and the fidelity job caught **real** existing defects)
  (W8/G25,G12,G24)
- Zero native measurement → criterion benches on the real invoke path + boot chain,
  ratio budgets, Stop-chain closure, nightly cold-start TTI (W4/G05)
- Perf pinned to two route ids → structural-density closure off ROUTES/subjects,
  marker-scale anti-vacuity, CI render/gzip baseline ratchet (W5/G13,C03,G30,G14)
- No memory/leak detection; plan probe only the owner-index query → CDP heap loop +
  effect-cleanup rule + per-DAL-shape EXPLAIN probe (W6/G15,G06)
- No duplication detection; unbounded DTO strings → jscpd Stop step + `.max()` sweep (W12/G17,G18)
- Coverage measures execution not assertion → mutation lane default-on (set-based
  ratchet, blocking incremental + nightly full) + assertion-presence Stop gate;
  mutation found a JWT audience bypass, a missing DELETE method, a 500-where-400,
  and a password-leaking redactor (W13/G09)
- Machinery exempt from the bar it enforces; canary proofs only existence-checked →
  data-driven complexity ratchet (re-lints with `--no-inline-config`), canary
  registry executes each proof, config-rule integrity check (W14/G16,G28)
- Release would ambush prior consumers → v0.1.5 update-skew leg, every ramp
  `NOTE`-only on pre-0.1.6 baseVersions, all new checks canaried (W15)

Deferred to 0.1.7 (recorded so they are not re-litigated):

- SLSA/provenance on the advertised `npx github:` path — needs an npm-publish /

- Forcing a `[corpus:]` cite at grouped decision sites that already have corpus

- A pinned-CPU runner class to tighten the CI-only cold-start budget from

## Process note — the adversarial method turned on the new work

Distinctive to this release: the audit method was re-run against v0.1.6's own
changes, and it repeatedly found real defects in code that had already passed
every gate. This is recorded because it is the strongest evidence for the
release's thesis (that "green" is not "correct" without a control that tries to
break the work):

- **The mutation lane found an auth bypass** in the shipped exemplar: a bare
  `API_AUDIENCE=` line yielded `''`, and jose's `if (audience && …)` skipped
  audience validation entirely — a token for a different API was ACCEPTED. Plus a
  DELETE method missing from CORS (delete never worked in the real app), a
  500-where-the-contract-promised-400, and a crash redactor leaking passwords.
- **The `mutation` module had never run**: StrykerJS was in neither package.json
  nor the catalog, so its `break: 80` threshold was unmeasured; when measured the
  exemplar scored 66.3%, and the module's nightly job would have exited 127.
- **Two review passes over W14 alone found nine defects** — including four
  pre-existing machinery-lint failures (the v0.1.5 "machinery obeys its own bar"
  claim was aspirational; four checks were red), a complexity ratchet the first
  fix left foolable by a same-named function crossing the limit, and a canary
  emptiness check that misjudged a real test titled after a `.mjs` file. Each was
  fixed and canary-proven; where a control cannot deliver its full promise (the
  spawn check proves a proof runs and is non-empty, not that it drives the gate
  red), the code and docs now say so rather than over-claim.

## Refuted claims (kept so they are not re-litigated)

From the scoping audit: the OpenAPI `createRoute` envelope IS machine-enforced by
the spec-walk (the "envelope is prose-only" framing was refuted); the exemplar's
`px-*`/`border-b` utilities are RTL-neutral, so the "ban physical-direction
utilities" lint was dropped (only two `text-right` numeric uses exist). From the
build-time reviews: the rule-integrity hashing fix held under adversarial retry;
the native-perf ratio design held (a dependency upgrade uniformly slowing the whole
invoke path is a recorded residual, Renovate-owned, not an agent green-turn).
